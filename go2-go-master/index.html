5. 快速开始

5.1 启动软件

Windows 用户

双击运行 launch_v4.bat 批处理文件，或在命令行中执行：

python go2_control_panel_v4.py

macOS / Linux 用户

在终端中执行：

python3 go2_control_panel_v4.py

软件主界面示意图

5.2 界面说明

软件启动后，您会看到一个分为三个区域的主窗口：

  • 左侧区域 - 动作选择面板
    • 列出了所有可用的动作
    • 按类别分组：基础、移动、舞蹈、特技
    • 点击动作可添加到右侧序列
  • 右上区域 - 网络设备列表
    • 显示局域网内发现的 GO2 设备
    • 勾选设备以建立连接
    • 显示设备 IP 地址和连接状态
  • 右中区域 - 舞蹈序列编辑器
    • 时间轴形式的序列列表
    • 显示已添加的动作序列
    • 可以调整动作顺序和参数
  • 右下区域 - 日志窗口
    • 实时显示操作日志
    • 显示执行状态和错误信息

5.3 打开模拟器（可选）

如果您没有真实的 GO2 设备，可以使用内置的模拟器进行测试：

6. 设备连接

6.1 扫描局域网设备

软件启动时会自动检测本机 IP 并填充扫描网段。点击"扫描网络"按钮开始扫描：

步骤 1：确认网段

检查自动填充的网段是否正确。例如：

  • 本机 IP: 192.168.1.100
  • 扫描网段: 192.168.1.0/24 ✅ 正确

步骤 2：开始扫描

点击"扫描网络"按钮，等待 30-60 秒。扫描过程中会实时显示进度：

扫描进度: 10% (25/254) 扫描进度: 20% (50/254) ... ✓ 发现GO2设备: 192.168.1.31 - 端口: [9991, 554]

步骤 3：查看结果

扫描完成后，设备列表会显示所有发现的设备。GO2 设备会自动勾选。

步骤 4：连接设备

勾选要连接的设备，点击"连接选中设备"按钮。

⚠️ 注意事项：
  • 确保 GO2 设备和电脑在同一局域网内
  • 确保设备已开机并联网
  • 如果设备未被发现，尝试手动 ping 设备 IP 检查网络连通性
  • 某些企业网络可能有端口限制，请联系网络管理员

6.2 手动添加设备

如果您知道设备的 IP 地址，可以直接手动添加：

在"手动添加 IP"输入框中输入设备 IP，例如：192.168.1.31

点击"添加设备"按钮

设备会出现在列表中，勾选后点击"连接选中设备"

7. 舞蹈序列编辑

舞蹈序列编辑器是本软件的核心功能，让您能够轻松创建复杂的机器狗表演动作。

7.1 添加动作到序列

在左侧动作列表中找到您想要的动作

双击动作或单击后点击"添加到序列"按钮

动作会出现在右侧的时间轴列表中

7.2 调整动作参数

每个动作都可以设置以下参数：

  • 动作名称：显示动作的中文名称
  • 动作时长：设置动作执行的持续时间（秒）
    • 基础动作通常 2-3 秒
    • 舞蹈动作通常 5-8 秒
    • 可以根据需要调整
  • 动作 ID：内部使用的标识符（自动生成）

7.3 调整序列顺序

您可以自由调整动作的执行顺序：

  • 拖拽排序：直接拖动动作项到新位置
  • 删除动作：选中动作后点击"移除"按钮或按 Delete 键
  • 清空序列：点击"清空序列"按钮删除所有动作
  • 保存序列：可以将当前序列保存到文件（功能开发中）

7.4 执行舞蹈序列

创建好序列后，点击"执行序列"按钮开始执行：

舞蹈序列执行示意图

  1. 选择执行模式
    • 🔴 真实设备执行：发送到已连接的真实 GO2 设备
    • 🟢 模拟器执行：仅在模拟器中演示
    • 🔶 混合模式：同时发送给真实设备和模拟器
  2. 观看执行过程
    • 进度条显示整体进度
    • 当前执行的动作会高亮显示
    • 日志窗口实时打印执行信息
  3. 执行完成
    • 显示成功/失败统计
    • 机器狗回到待机状态
💡 使用技巧：
  • 先用模拟器测试序列，确认效果后再用真实设备
  • 合理安排动作时长，让动作衔接更自然
  • 可以在序列开始前添加"站立"动作，确保机器狗处于正确姿态
  • 在序列结束后添加"蹲下"或"停止"动作，让机器狗平稳结束

8. 模拟器使用

GO2 AIR 模拟器是一个独立的可视化组件，让您在没有真实设备的情况下也能测试和预览动作。

8.1 模拟器界面

模拟器界面组成

模拟器包含以下部分：

  • 模拟狗显示区域：显示每只模拟狗的可视化状态
    • 圆形背景：颜色表示当前动作类型
    • 狗图标：简化的机器狗图形
    • 进度条：显示动作执行进度
    • 状态文本：显示当前动作名称
  • 控制面板：
    • ➕ 添加模拟狗：增加新的模拟狗
    • ➖ 移除选中：删除选中的模拟狗
    • 🗑️ 清空全部：删除所有模拟狗
  • 快速测试按钮：
    • 预设的动作按钮，点击即可测试
    • 包括：站立、蹲下、打招呼、前进、舞蹈、跳跃等
  • 执行日志：
    • 记录所有执行过的动作
    • 显示时间戳和详细信息

8.2 选择要控制的模拟狗

每只模拟狗都有一个复选框：

  • 勾选：该模拟狗会被包含在执行中
  • 不勾选：该模拟狗不会执行动作
💡 提示：您可以创建多只模拟狗，然后选择其中一部分执行特定动作，实现分组控制效果！

8.3 颜色映射系统

模拟器使用颜色来区分不同类型的动作：

颜色 动作类型 示例
绿色 基础动作 站立、蹲下
蓝色 姿态调整 伸展、Sit
黄色 交互动作 打招呼、握手
紫色 移动动作 前进、后退、转向
粉色 舞蹈动作 Dance1-5
红色 特技动作 跳跃、翻滚

8.4 模拟器与主程序配合

模拟器可以独立运行，也可以与主程序配合使用：

  1. 启动主程序：运行 go2_control_panel_v4.py
  2. 打开模拟器：点击主程序中的"🟢 打开模拟器"按钮
  3. 创建序列：在主程序中添加动作到序列
  4. 选择执行模式：在执行对话框中选择"模拟器执行"
  5. 观看效果：模拟器会实时显示动作执行过程

10. 常见问题

Q1: 扫描网络时找不到设备怎么办？

A:请按以下步骤排查：

  1. 确认 GO2 设备和电脑在同一局域网
  2. 检查设备是否开机并联网（指示灯正常）
  3. 手动 ping 设备 IP：打开命令行，执行 ping 192.168.x.x
  4. 检查防火墙设置，确保允许 Python 网络访问
  5. 尝试手动添加设备 IP
  6. 如果是企业网络，可能需要联系网络管理员开放端口
Q2: 模拟器没有反应怎么办？

A:可能的原因和解决方法：

  • 未勾选模拟狗：确保至少勾选了一只模拟狗的复选框
  • 模拟器窗口未打开：点击"🟢 打开模拟器"按钮
  • 动作名称不匹配：某些动作可能只支持英文名称或中文名称
  • 信号未连接：重启模拟器和主程序
Q3: 执行序列时机器狗没有反应？

A:请检查：

  1. 确认已勾选并连接了设备（设备列表中显示为绿色✅）
  2. 查看日志窗口，检查是否有错误信息
  3. 确认选择了正确的执行模式（真实设备执行）
  4. 检查网络连接是否稳定
  5. 尝试单个动作测试，确认设备正常
Q4: 软件启动报错 "ModuleNotFoundError" 怎么办？

A:这说明缺少依赖库，请安装：

pip install PyQt5 go2-webrtc-driver

如果安装速度慢，使用国内镜像：

pip install -i https://pypi.tuna.tsinghua.edu.cn/simple PyQt5 go2-webrtc-driver
Q5: 如何控制多只机器狗同时执行？

A:很简单！

  1. 在扫描网络后，勾选多台设备
  2. 点击"连接选中设备"
  3. 创建舞蹈序列
  4. 执行时选择"真实设备执行"
  5. 所有已连接的设备会同时执行相同的动作

提示：您也可以通过不勾选某些设备来实现选择性控制！

Q6: 可以保存和加载舞蹈序列吗？

A:当前版本（v4.2）的保存/加载功能正在开发中。暂时建议：

  • 截图记录您的序列配置
  • 或用笔记记录动作顺序和参数
  • 后续版本会添加完整的保存/加载功能
