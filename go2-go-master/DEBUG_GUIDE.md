## 🔍 常见问题排查

### 问题 1: 模拟器没有反应

**检查项**：
```
1. 查看是否有 "[模拟器] 收到动作" 输出
   - 没有 → 信号未发送，检查信号连接
   - 有 → 继续

2. 查看是否有 "[模拟器] 当前选中的狗"
   - 空列表 [] → 没有勾选模拟狗
   - 有狗 → 继续

3. 查看是否有 "[模拟器] X 开始执行"
   - 没有 → 颜色匹配失败
   - 有 → 界面应该有变化
```

**解决方案**：
```python
# 确保勾选了模拟狗的复选框
# 检查动作名称是否在颜色映射中
```

### 问题 2: 真实设备没有响应

**检查项**：
```
1. 查看是否有 "[序列执行] 🔴 发送到真实设备"
   - 没有 → 没有选择真实设备执行
   - 有 → 继续

2. 查看 API ID 是否正确
   - 不正确 → 检查 SPORT_CMD 映射
   - 正确 → 继续

3. 查看是否有 "[序列执行] ✓ GO2-1 发送成功"
   - 没有成功 → 检查网络连接
   - 成功 → 检查设备是否收到命令
```

**解决方案**：
```python
# 先测试单个动作
# 检查设备是否真的连接
# 查看控制台的 GO2 库日志
```

### 问题 3: 动作名称不匹配

**现象**：
```
[模拟器] 收到动作: StandUp  # 英文ID
[模拟器] 没有匹配到颜色
```

**原因**：
- 动作 ID 和动作名称不对应

**解决方案**：
```python
# 在模拟器中添加更多映射
color_map = {
    "StandUp": QColor(...),
    "站立": QColor(...),
    # 两者都支持
}
```

### 问题 4: 信号未连接

**现象**：
```
[序列执行] 🟢 发送到模拟器...
# 但没有模拟器的输出
```

**原因**：
- 模拟器窗口未打开
- 信号未连接

**解决方案**：
```python
# 1. 确保先打开模拟器
# 2. 检查信号连接代码
self.dance_thread.simulator_action.connect(
    self.simulator_window.simulator.execute_action
)
```

## 🎯 完整调试流程

### Step 1: 启动程序
```bash
cd C:\Users\Timer's\unitree-dance-qt

```
点击 "🟢 打开模拟器" 按钮

点击 "执行序列"

### Step 5: 分析输出

**正常输出应该是**：
```
[主控制面板] 执行模式: 模拟器
[主控制面板] 使用模拟器: True
↓
[序列执行] 使用模拟器: True
↓
[序列执行] 🟢 发送到模拟器...
↓

```

### Step 6: 定位问题

根据缺失的输出定位问题：

| 缺失的输出 | 问题位置 | 检查内容 |
|-----------|---------|---------|
| [主控制面板] 执行模式 | execute_sequence | 对话框是否正确返回 |
| [序列执行] 使用模拟器 | DanceSequenceThread | use_simulator 参数 |
| [序列执行] 🟢 发送 | if self.use_simulator | 条件判断 |
| [模拟器] 收到动作 | execute_action | 信号连接 |
| [模拟器] 匹配颜色 | color_map | 动作名称映射 |

## 📝 调试技巧

### 技巧 1: 缩小范围

```
不要直接测试完整序列
→ 先测试单个动作
→ 再测试 2 个动作
→ 最后测试完整序列
```

### 技巧 2: 分离测试

```
1. 先测试模拟器单独运行
   python go2_simulator.py
   
2. 再测试主程序单独运行
   python go2_control_panel_v4.py
   
3. 最后测试集成
```

### 技巧 3: 观察界面

```
控制台输出 → 逻辑层
界面变化 → 显示层

都要检查！
```

### 技巧 4: 对比验证

```
模拟器模式 vs 真实模式
相同点：序列流程
不同点：发送目标

对比输出找出差异
```
