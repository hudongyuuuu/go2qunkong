## 🚀 快速开始

### 什么是动作脚本？

动作脚本是一个JSON格式的文件，用于记录机器狗的动作序列。每个动作包含：
- **robot_id**: 目标机器狗ID
- **action_type**: 动作类型
- **params**: 动作参数（可选）
- **timestamp**: 动作执行时间点（毫秒）
- **duration**: 动作持续时长（秒）

### 最简单的脚本示例

```json
{
  "name": "简单问候",
  "description": "机器人站立并挥手致意",
  "actions": [
    {
      "robot_id": "robot_001",
      "action_type": "stand",
      "params": {},
      "timestamp": 0,
      "duration": 1.5
    },
    {
      "robot_id": "robot_001",
      "action_type": "wave_hand",
      "params": {},
      "timestamp": 2000,
      "duration": 3.0
    }
  ],
  "created_at": "2024-01-15T14:30:00",
  "total_duration": 5000
}
```

**解释**：
- 0秒时，robot_001 执行"站立"动作，持续1.5秒
- 2秒后，robot_001 执行"挥手"动作，持续3秒
- 总时长：5秒

## 📄 脚本文件格式

### 完整脚本结构

```json
{
  "name": "脚本名称",
  "description": "脚本描述（可选）",
  "actions": [
    {
      "robot_id": "机器人ID",
      "action_type": "动作类型",
      "params": {
        "参数名": "参数值"
      },
      "timestamp": 时间戳(毫秒),
      "duration": 时长(秒)
    }
  ],
  "created_at": "创建时间(ISO格式)",
  "total_duration": 总时长(毫秒)
}
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| name | string | ✅ | 脚本名称，文件保存时会使用此名称 |
| description | string | ❌ | 脚本描述，说明脚本用途 |
| actions | array | ✅ | 动作列表 |
| created_at | string | ❌ | 创建时间，自动生成 |
| total_duration | number | ✅ | 总时长，自动计算为最后一个动作的timestamp |

### 动作对象说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| robot_id | string | ✅ | 目标机器狗的唯一标识 |
| action_type | string | ✅ | 动作类型，参见[内置动作列表](#内置动作列表) |
| params | object | ❌ | 动作参数，默认为{} |
| timestamp | number | ✅ | 动作执行时间点（毫秒），从0开始 |
| duration | number | ❌ | 动作持续时长（秒），用于时间轴编辑 |

### 动作参数说明

大多数动作不需要参数（params为空）。某些复杂动作可以接受可选参数：

```json
{
  "action_type": "spin",
  "params": {
    "angle": 720  // 旋转720度（两圈）
  }
}
```

常见参数：
- `angle`: 旋转角度（spin动作）
- `speed`: 速度系数（移动动作，0.5-2.0）
- `height`: 跳跃高度（jump动作）
- `repeat`: 重复次数（舞蹈动作）

## 🎬 录制与编辑

### 方式1：实时录制

1. 在"动作控制"页面，选择目标机器狗
2. 切换到"脚本编辑"标签页
3. 输入脚本名称
4. 点击"开始录制"
5. 在动作列表中双击动作，或选择动作后点击"录制到脚本"
6. 完成后点击"停止录制"
7. 点击"保存脚本"

**优点**：直观、简单
**适用**：快速创建简单脚本

### 方式2：手动编辑JSON

1. 使用任何文本编辑器打开脚本文件
2. 修改actions数组
3. 保存文件
4. 在软件中点击"刷新列表"或重新加载

**优点**：精确控制时间点
**适用**：需要精确同步多个机器狗

### 方式3：导入/导出

**导出脚本**：
1. 加载或创建脚本
2. 点击"导出脚本"按钮
3. 选择保存位置

**导入脚本**：
1. 点击"从文件加载"按钮
2. 选择JSON脚本文件
3. 脚本自动添加到脚本库

## 👥 群组控制

### 单狗控制

脚本中只指定一个robot_id：

```json
{
  "name": "单狗表演",
  "actions": [
    {"robot_id": "robot_001", "action_type": "stand", "timestamp": 0},
    {"robot_id": "robot_001", "action_type": "jump", "timestamp": 2000}
  ]
}
```

### 多狗同步（相同动作）

多只机器狗在同一时间执行相同动作：

```json
{
  "name": "同步表演",
  "actions": [
    {"robot_id": "robot_001", "action_type": "stand", "timestamp": 0},
    {"robot_id": "robot_002", "action_type": "stand", "timestamp": 0},
    {"robot_id": "robot_003", "action_type": "stand", "timestamp": 0},
    {"robot_id": "robot_001", "action_type": "wave_hand", "timestamp": 2000},
    {"robot_id": "robot_002", "action_type": "wave_hand", "timestamp": 2000},
    {"robot_id": "robot_003", "action_type": "wave_hand", "timestamp": 2000}
  ]
}
```

### 多狗协同（不同动作）

多只机器狗在同一时间执行不同动作：

```json
{
  "name": "协同表演",
  "actions": [
    {"robot_id": "robot_001", "action_type": "wave_hand", "timestamp": 0, "duration": 3.0},
    {"robot_id": "robot_002", "action_type": "bow", "timestamp": 0, "duration": 3.5},
    {"robot_id": "robot_003", "action_type": "jump", "timestamp": 0, "duration": 2.5},
    {"robot_id": "robot_001", "action_type": "dance_move_1", "timestamp": 4000},
    {"robot_id": "robot_002", "action_type": "dance_move_2", "timestamp": 4000},
    {"robot_id": "robot_003", "action_type": "dance_move_3", "timestamp": 4000}
  ]
}
```

### 多狗时序编排

为不同机器狗安排不同时间点的动作：

```json
{
  "name": "波浪表演",
  "actions": [
    {"robot_id": "robot_001", "action_type": "jump", "timestamp": 0},
    {"robot_id": "robot_002", "action_type": "jump", "timestamp": 500},
    {"robot_id": "robot_003", "action_type": "jump", "timestamp": 1000},
    {"robot_id": "robot_004", "action_type": "jump", "timestamp": 1500},
    {"robot_id": "robot_005", "action_type": "jump", "timestamp": 2000}
  ]
}
```

**效果**：形成波浪式跳跃效果

### 选择性群控

在UI界面中：
1. 在"动作控制"页面的"选择机器人"下拉框中选择
2. 录制时会为选中的机器人记录动作
3. 执行时只向脚本中指定的robot_id发送指令

## ⏱️ 时间轴编辑

### 时间戳计算

时间戳单位：**毫秒 (ms)**

计算公式：
```
timestamp = 前一动作timestamp + 动作间隔时间 × 1000
```

**示例**：
```
动作1: timestamp = 0       (0秒)
动作2: timestamp = 2000    (2秒后)
动作3: timestamp = 5000    (5秒后)
动作4: timestamp = 7000    (7秒后)
```

### 动作间隔建议

| 场景 | 建议间隔 | 说明 |
|------|----------|------|
| 基础动作切换 | 1000-2000ms | 确保前一个动作完成 |
| 连续移动 | 500-1000ms | 保持流畅 |
| 舞蹈动作 | 3000-5000ms | 配合动作时长 |
| 机器狗协同 | 0-500ms | 制造波浪效果 |
| 不同机器狗 | 0ms | 同时执行 |

### 使用预估时长

每个动作都有预估时长（duration字段），用于：
- 时间轴编辑器显示动作块长度
- 计算下一个动作的开始时间
- 避免动作重叠

**建议**：
- 下一个动作的timestamp ≥ 前一动作的timestamp + 前一动作的duration × 1000
- 或：下一动作timestamp = 上一动作timestamp + 上一动作duration × 1000 + 间隔

**示例**：
```json
{
  "actions": [
    {"action_type": "stand", "timestamp": 0, "duration": 1.5},
    {"action_type": "jump", "timestamp": 2500, "duration": 2.5},
    {"action_type": "bow", "timestamp": 6000, "duration": 3.5}
  ]
}
```

计算：
- stand: 0ms开始，持续1500ms，1500ms完成
- jump: 1500 + 1000(间隔) = 2500ms开始
- bow: 2500 + 2500 + 1000(间隔) = 6000ms开始

## ❓ 常见问题

### Q1: 脚本执行失败怎么办？

**检查清单**：
1. 机器狗是否已连接？
2. robot_id是否正确？
3. action_type是否拼写正确？
4. 机器狗是否有足够电量？
5. 网络连接是否正常？

**解决方法**：
- 查看日志输出
- 逐个测试每个动作
- 检查机器狗状态面板

### Q2: 动作执行不准确？

**可能原因**：
- 网络延迟
- timestamp间隔太小
- 机器狗负载过高

**优化建议**：
- 增加动作间隔时间（至少500ms）
- 使用有线网络而非WiFi
- 减少同时执行的机器狗数量

### Q3: 如何创建复杂的群控表演？

**步骤**：
1. 先为每个机器狗单独编写动作序列
2. 确定各个机器狗动作的相对时间关系
3. 调整timestamp实现协同效果
4. 逐个测试每个机器狗的动作
5. 整体测试并微调时间

**技巧**：
- 使用表格规划时间轴
```
时间    robot_001    robot_002    robot_003
0s      stand        stand        stand
2s      wave_hand    bow          jump
5s      dance1       dance2       dance3
```

### Q4: 脚本文件保存在哪里？

脚本文件默认保存在：
```
C:\Users\Timer's\unitree-dance-qt\scripts\
```

文件名格式：`脚本名称.json`

### Q5: 如何备份和分享脚本？

**备份**：
- 复制 `scripts/` 文件夹
- 或使用"导出脚本"功能

**分享**：
- 导出脚本JSON文件
- 发送给其他人
- 对方使用"从文件加载"导入

### Q6: 预估时长不准确怎么办？

预估时长仅供参考，实际时长可能因：
- 机器狗型号
- 地面摩擦力
- 电量水平
- 负载情况

**调整方法**：
- 手动编辑duration字段
- 根据实际情况测试调整
- 在脚本注释中记录实测时长
