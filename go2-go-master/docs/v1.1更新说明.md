# Unitree GO2 机器狗群控 - v1.1 更新说明

## 🎉 新增功能

本次更新完成了以下功能：

### 1. ✅ 动作预估时间长度

为所有33个内置动作添加了预估时长，便于时间轴编辑和动作编排。

**更新内容**：
- 每个动作包含 `duration` 字段（单位：秒）
- 新增 `get_action_duration()` 方法自动获取动作时长
- 录制脚本时自动记录动作时长

**动作时长分类**：
- 基础动作: 1.5-2.5秒
- 移动: 2.5-5.0秒
- 转向: 2.0-3.0秒
- 特殊动作: 2.5-4.5秒
- 舞蹈动作: 4.0-6.0秒
- 全身动作: 3.0-8.0秒

**使用方法**：
```python
# 获取动作时长
duration = script_service.get_action_duration("wave_hand")  # 返回 3.0 秒
```

### 3. ✅ 选择性群控功能

实现了多机器狗选择和控制功能，满足不同场景需求。

**功能特点**：
- **多选控制**: 在"动作控制"页面可同时选择多台机器狗
- **一键全选/清空**: 方便快速操作
- **实时计数**: 显示已选择的机器狗数量
- **连接状态显示**: 每个复选框旁边显示连接状态（🟢已连接 / 🔴未连接）

**使用场景**：

**场景1: 单机操作**
- 勾选一台机器狗
- 执行动作，只有选中的机器狗执行

**场景2: 同步表演**
- 勾选多台机器狗
- 执行动作，所有选中的机器狗同步执行相同动作

**场景3: 分组控制**
- 按需选择不同的机器狗组合
- 为不同组录制不同动作
- 实现复杂的群控编排

**操作流程**：
1. 在"选择目标机器狗"区域勾选目标机器狗
2. 选择要执行的动作
3. 点击"执行选中动作"或"录制到脚本"
4. 查看日志确认执行结果

**日志输出示例**：
```
✓ 执行动作: 挥手 -> Robot 1 (robot_001)
✓ 执行动作: 挥手 -> Robot 2 (robot_002)
✓ 执行动作: 挥手 -> Robot 3 (robot_003)
动作执行完成: 成功 3 台
```

```json

      "robot_id": "robot_002",
      "action_type": "stand",
      "timestamp": 0,
      "duration": 1.5
    },
    {
      "robot_id": "robot_003",
      "action_type": "stand",

      "duration": 1.5

      "robot_id": "robot_001",
      "action_type": "wave_hand",

    {

      "action_type": "bow",

      "robot_id": "robot_003",
      "action_type": "jump",
```

5. 选择"挥手"动作，点击"录制到脚本"
6. 停止录制并保存

## 🎯 实际应用场景

### 1. 队形表演
- 选择前排机器狗执行挥手动作
- 选择后排机器狗执行跳跃动作
- 形成前后对比的视觉效果

### 2. 轮流展示
- 依次选择不同机器狗执行特技动作
- 每台机器狗单独展示

### 3. 协同舞蹈
- 所有机器狗同步执行基础动作
- 部分机器狗执行特殊动作作为亮点
- 通过选择性控制实现层次感

### 4. 分组训练
- 将机器狗分成不同组
- 每组练习不同的动作序列
- 便于教学和训练

## 🔄 迁移指南

### 从旧版本升级

如果您之前使用的是单机控制版本：

1. **添加机器狗**：
   - 在左侧"添加机器人"表单中添加所有机器狗
   - 等待连接状态变为 🟢

2. **选择目标**：
   - 在"选择目标机器狗"区域勾选要控制的机器狗
   - 使用"全选"快速选中所有

3. **执行动作**：
   - 选择动作后点击"执行选中动作"
   - 所有选中的机器狗会同时执行

4. **录制脚本**：
   - 开始录制
   - 执行动作时会为所有选中的机器狗录制
   - 可以改变选择来为不同机器狗录制不同动作

## 💡 使用建议

1. **网络优化**：
   - 使用有线网络连接减少延迟
   - 确保路由器性能足够支持多设备

2. **电量管理**：
   - 执行前检查所有机器狗电量
   - 建议电量 > 50% 时进行群控

3. **动作间隔**：
   - 连续动作建议间隔 1-2秒
   - 给机器狗足够时间完成前一个动作

4. **测试顺序**：
   - 先单机测试
   - 再少机群控（2-3台）
   - 最后多机协同（5台以上）
