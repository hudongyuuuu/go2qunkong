# GO2 控制面板 v4 - 舞蹈增强版（固件 1.1.11 适配）

## ⚠️ 固件兼容性（重要）

本程序通过 **WiFi (WebRTC) 局域网模式** 连接 GO2，对固件版本有要求：

- ✅ **适用固件：1.1.11**（`go2_control_panel_v4-1.1.11.py` 就是针对该版本固件适配的）
- ❌ **不适用固件：1.1.15 及以上**

宇树在 1.1.15 版本固件中改动了连接机制，**WiFi 局域网连接模式已无法使用**。如果你的 GO2 已升级到 1.1.15，本程序当前无法通过 WiFi 连接设备。

> 连接前请先在 App 中确认机器狗的固件版本。如需使用本程序，请保持固件在 1.1.11。

## 🎉 本版改进（go2_control_panel_v4-1.1.11.py）

相比旧版 `go2_control_panel_v4.py`，本版针对固件 1.1.11 做了以下改进：

### 1. 🔌 修复 1.1.11 固件连接
- **修复**：1.1.11 版本的机器狗无法连接的问题
- 使用 `unitree_webrtc_connect`（2.x）建立 WiFi (WebRTC) 局域网连接

### 2. 🐾 舞蹈序列并行动作
- 新增**并行动作组**：多台机器狗可以**同时**执行各自不同的动作
- 序列编辑器支持「➕ 添加并行」，也可把已选中的多个动作「🔗 合并为并行」/「✂️ 拆分并行组」

### 3. 🎯 每个动作单独选择执行的狗
- 序列中的每一步都可以单独指定**由哪些狗执行**
- 添加并行动作时，可为每只狗分别选择动作（不需要的狗留空即可）

### 4. 🛠️ 动作修复与更新
- **修复**：部分动作无法执行的 bug
- 新增 **Rest（休止）** 动作
- 更新、调整了部分内置动作

## ⚠️ 系统要求

- **Python**: 3.11+
- **库**: unitree_webrtc_connect（2.x）
- **机器狗**: Unitree GO2 Air
- **固件**: 1.1.11（WiFi 局域网连接模式需要；1.1.15 及以上不支持）

## 🐛 已知问题

- ❌ **无法切换步态**（步态类动作暂不生效）
- ⚠️ **舞蹈序列中偶尔跳过动作**：并行组里不要一次写太多动作一块执行，建议写一个执行一下
- 部分高难度动作需要机器狗本身支持，地面需平坦防滑

## 💡 提示

1. **首次使用**：先在开阔空间测试基础动作
2. **运动指令**：确保机器狗先站立（StandUp），运动动作需 normal 模式
3. **舞蹈序列**：建议从短序列开始（5-10 个动作）
4. **并行组**：一个并行组里别塞太多动作，宁可拆成多个组逐个执行
5. **随时停止**：所有执行都可以中断
