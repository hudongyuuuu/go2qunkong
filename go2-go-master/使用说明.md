# GO2 机器狗控制面板 - 使用说明

## ✅ 已完成的修改

### 1. 改进的扫描功能

**修改前：**
- 只扫描有 RTSP 端口 554 的设备
- 找不到真正的机器狗（192.168.71.31，端口 9991）

**修改后：**
- ✅ 扫描所有在线设备
- ✅ 检查多个端口：554, 5000, 8081, 8888, 8899, 9991
- ✅ 显示每个设备的开放端口
- ✅ 智能标记可能的 GO2 设备
- ✅ 用蓝色背景高亮显示可能的 GO2 设备

### 2. 设备识别规则

程序会根据开放端口判断是否是 GO2 机器狗：
- **有 8081 或 9991 端口** → 很可能是 GO2（WebRTC 控制）
- **有 554 端口** → 可能是 GO2（RTSP 视频流）
- **标记为 [GO2?] 并用蓝色背景显示**

## 🚀 使用步骤

### 步骤 1：启动程序

```bash
cd "C:\Users\Timer's\unitree-dance-qt"
python go2_control_panel_full.py
```

### 步骤 2：扫描局域网

点击 **"🔍 扫描局域网"** 按钮

等待扫描完成（约 3-5 秒），会显示：

### 步骤 3：识别机器狗

**根据手机 APP 显示的 IP（192.168.71.31），在列表中找到对应的设备：**

示例显示：
```
设备-1: 192.168.71.1 | 端口:
设备-2: 192.168.71.10 | 端口: 554, 5000 [GO2?]
设备-3: 192.168.71.31 | 端口: 9991 [GO2?]  ← 这是你真正的机器狗
设备-4: 192.168.71.33 | 端口:
```

### 步骤 4：测试连接

1. **选择正确的设备**（192.168.71.31）
2. 点击 **"测试连接"** 按钮
3. 查看连接测试结果：
   - Ping 测试
   - RTSP 端口测试（如果有）
   - HTTP 端口测试（如果有）

### 步骤 5：连接机器狗（需要修复 GO2 库）

**重要：** 目前 GO2 库还有 "Optional" 错误，需要先修复：

#### 方法 1：使用管理员 PowerShell

1. 以管理员身份打开 PowerShell
2. 运行：
```powershell
$file = "C:\ProgramData\Miniconda3\lib\site-packages\go2_webrtc_driver\util.py"
$content = Get-Content $file -Raw -Encoding UTF8

# 添加 Optional 导入
if ($content -match 'from typing import') {
    $content = $content -replace 'from typing import ([^\r\n]+)', 'from typing import $1, Optional'
}

# 替换类型注解
$content = $content -replace 'str \| None', 'Optional[str]'
$content = $content -replace 'dict \| None', 'Optional[dict]'

# 保存
$content | Set-Content $file -Encoding UTF8 -NoNewline
```

#### 方法 2：手动编辑

1. 以管理员身份打开记事本
2. 打开文件：`C:\ProgramData\Miniconda3\lib\site-packages\go2_webrtc_driver\util.py`
3. 找到第 6-7 行左右：
```python
from typing import Dict
```
4. 改为：
```python
from typing import Dict, Optional
```
5. 找到所有 `str | None`，替换为 `Optional[str]`
6. 保存文件

修复完成后，重新运行控制面板即可连接机器狗。

## 📝 当前已知问题

### 1. GO2 库的 "Optional" 错误
- **原因：** Python 3.9 不支持 `str | None` 语法
- **解决：** 按照上述方法修复 util.py 文件

### 2. WebRTC 连接失败
- **可能原因：**
  - 机器狗未与 APP 配对
  - 防火墙阻止
  - 需要先在 APP 中激活 WebRTC 功能
